## 1. 한 문장 요약

SAM 2는 Hiera가 만든 현재 프레임 feature를 최근 예측과 사용자가 prompt한 프레임의 공간 메모리, 그리고 object pointer에 cross-attention한 뒤 SAM식 mask decoder로 분할하고 그 결과를 다시 메모리에 쓰는 streaming architecture로, 이미지 분할과 interactive video object segmentation을 하나의 모델로 통합한다.

## 2. SAM에서 SAM 2로 확장되는 문제

이미지는 한 시점만 표현하지만 영상의 객체는 다음 변화를 겪는다.

- 이동, 변형, 크기 변화
- 카메라 이동과 motion blur
- 조명 변화와 낮은 화질
- 다른 객체에 의한 가림과 재등장
- 비슷하게 생긴 여러 객체 사이의 identity 혼동
- 수백 또는 수천 프레임에 걸친 장기 추적

이미지 SAM과 별도 tracker를 직렬로 연결할 수는 있다. 하지만 tracker가 틀린 중간 프레임에서 SAM으로 마스크를 다시 만든 뒤 tracking을 재시작해야 하고, 이전 interaction의 맥락을 단일 모델이 사용하지 못한다. SAM 2는 분할과 propagation, correction을 하나의 memory-conditioned 모델 안에 넣는다.

## 3. Promptable Visual Segmentation task

PVS는 이미지의 Segment Anything task를 시공간으로 확장한다. 영상 `V={I_1,...,I_T}`의 임의 프레임에 positive/negative click, box, mask를 넣으면 관심 객체의 시간별 마스크 집합인 masklet을 예측한다.

```text
prompts on arbitrary frames + video
  -> {M_1, M_2, ..., M_T}
```

사용자는 어떤 프레임에서든 추가 prompt로 결과를 고칠 수 있다. 모델은 그 프레임에 즉시 마스크를 응답하고, correction을 다른 프레임으로 전파해야 한다.

기존 과제와의 관계는 다음과 같다.

- 이미지 SA: 길이 1인 video의 PVS
- semi-supervised VOS: 첫 프레임에 ground-truth mask 하나만 주는 PVS
- interactive VOS: 여러 프레임에 제한된 형태의 prompt를 주는 PVS

PVS는 category label이 아니라 시각적으로 경계가 정의되는 **valid object**를 추적한다. 부분과 하위 부분도 대상이며, 경계가 시각적으로 정의되지 않는 추상 영역은 범위 밖이다.

## 4. 전체 아키텍처

한 객체에 대한 프레임 `t`의 흐름은 다음과 같다.

```text
frame I_t
  -> Hiera image encoder + FPN
  -> unconditioned image embedding E_t

E_t + previous spatial memories + object pointers
  -> 4-layer memory attention
  -> memory-conditioned frame feature Z_t

Z_t + current points/box/mask
  -> SAM-like prompt encoder and mask decoder
  -> mask M_t, predicted IoU q_t, presence o_t, object pointer p_t

E_t + downsample(M_t)
  -> memory encoder
  -> spatial memory R_t

R_t, p_t
  -> bounded memory bank
  -> used by later propagation or refinement
```

정적 이미지에서는 memory bank가 비어 있으므로 memory-conditioned video tracker가 아니라 SAM과 유사한 prompt segmenter로 동작한다.

## 5. Image encoder와 multi-scale feature

### 5.1 Hiera backbone

SAM 2는 MAE로 사전학습한 hierarchical Hiera encoder를 사용한다. T, S, B+, L 크기를 지원하며 기본 실험은 Hiera-B+다. 계층형 표현 덕분에 mask decoder에 고해상도 skip feature를 제공할 수 있다.

Hiera의 stage 출력은 다음처럼 쓰인다.

```text
Stage 1, stride 4  ---- high-resolution skip --> mask decoder
Stage 2, stride 8  ---- high-resolution skip --> mask decoder
Stage 3, stride 16 --+
                     +--> FPN --> [B,256,64,64] at 1024 input
Stage 4, stride 32 --+
```

stride 4와 8 feature는 memory attention을 우회하고 decoder upsampling layer에 더해져 경계 세부를 복원한다. stride 16과 32만 FPN으로 융합해 memory attention의 기본 image embedding을 만든다.

### 5.2 위치 인코딩과 attention kernel

Hiera에는 windowed absolute positional embedding을 사용한다. 이전 설계의 relative positional bias(RPB)는 image encoder에서 모두 제거하고, global position embedding을 보간해 window 사이 위치 정보를 전달한다. RPB 제거는 `1024 x 1024`에서 FlashAttention-2를 사용하기 쉽게 해 속도를 높인다.

global attention은 모든 block에 넣지 않고 일부 block에만 둔다. 논문 설정의 block index는 다음과 같다.

- Hiera-T: 5, 7, 9
- Hiera-S: 7, 10, 13
- Hiera-B+: 12, 16, 20
- Hiera-L: 23, 33, 43

이 숫자는 stage 수가 아니라 global attention을 적용하는 block 위치다.

## 6. Memory attention

### 6.1 역할

현재 image embedding `E_t`는 아직 어느 객체를 추적하는지 모른다. memory attention은 과거 예측, prompted frame, object pointer를 읽어 관심 객체에 조건화된 `Z_t`를 만든다.

기본 4개 Transformer layer의 각 layer는 다음 순서를 따른다.

```text
X <- SelfAttention(X) + X
X <- CrossAttention(query=X,
                    key/value=spatial memories + object pointers) + X
X <- MLP(X) + X
```

현재 frame feature끼리 self-attention하고, memory bank에 cross-attention한 다음 MLP를 적용한다. 논문은 복잡한 custom memory operator 대신 vanilla attention을 사용해 효율적인 attention kernel의 이점을 얻는다.

### 6.2 2D RoPE

memory attention의 self-attention과 cross-attention에는 sinusoidal absolute position과 2D Rotary Positional Embedding을 사용한다. spatial correspondence가 없는 object pointer token에는 RoPE를 적용하지 않는다.

ablation에서 image encoder RPB를 제거하고 memory attention에 2D RoPE를 넣은 기본 설정은 9 zero-shot J&F 71.5로, RPB만 사용한 설정 70.7과 position encoding을 모두 단순화한 설정 70.3보다 높았다. 속도는 512 입력의 상대값으로 각각 0.93, 1.00, 1.04였지만 논문은 1024에서는 RPB 제거로 FlashAttention-2를 쓸 수 있어 차이가 작아진다고 설명한다.

### 6.3 엄밀한 causal model은 아니다

online streaming에서는 현재 시점까지 생성된 memory만 읽으므로 동작상 causal하다. 그러나 PVS는 임의 프레임에 correction을 허용하고 prompted frame이 현재 처리 방향 기준으로 미래에 있을 수도 있다. offline 재전파에서는 그 prompted memory를 사용할 수 있다. 따라서 아키텍처를 언어 모델식 고정 causal mask로 이해하기보다, **현재 memory bank에 들어 있는 항목만 key/value로 읽는 조건부 attention**으로 이해하는 편이 정확하다.

## 7. Prompt encoder와 mask decoder

### 7.1 SAM에서 유지한 부분

점과 박스는 위치 인코딩과 prompt type embedding의 합으로 sparse token이 되고, mask prompt는 convolution으로 dense embedding이 된다. two-way Transformer가 prompt token과 frame embedding을 양방향으로 갱신하고 dynamic mask head가 마스크를 생성한다.

```text
conditioned embedding: [B,256,64,64]
  -> two-way Transformer
  -> transposed convolution upsampling
  -> per-mask dynamic dot product
  -> low-resolution mask logits
```

SAM 2는 Hiera stride 4, 8 skip feature를 upsampling 중에 더한다는 점이 원본 SAM과 다르다.

### 7.2 영상의 모호성

한 프레임에서 자전거 타이어만 보이면 click이 타이어를 뜻하는지 자전거 전체를 뜻하는지 미래 프레임을 보기 전에는 결정하기 어렵다. SAM 2는 각 단계에서 여러 마스크를 예측한다. 추가 prompt가 모호성을 해소하지 않으면 현재 프레임 predicted IoU가 가장 높은 후보 하나만 다음 프레임으로 propagate한다.

이 정책은 계산을 제어하지만 한 번 잘못 고른 후보가 memory encoder를 거쳐 미래 memory가 되는 error propagation 위험도 있다.

### 7.3 Object presence head

영상에서는 관심 객체가 가려져 현재 프레임에 존재하지 않을 수 있다. SAM과 달리 SAM 2는 occlusion/presence token과 MLP head를 추가해 객체가 보이는지 예측한다.

```text
occlusion token -> MLP -> visibility score
```

보이지 않는다고 판단한 frame memory에는 learned occlusion embedding을 더한다. 이를 통해 빈 마스크를 단순한 segmentation failure와 구분하려 한다.

### 7.4 Object pointer

선택된 mask의 output token은 그 프레임의 object pointer가 된다. 원래 256차원 pointer를 memory channel에 맞춰 64차원 token 네 개로 나눠 memory cross-attention의 key/value에 넣는다.

```text
p_t in R^256 -> [p_t^1, p_t^2, p_t^3, p_t^4], each in R^64
```

spatial memory가 위치와 형태를 담는다면 pointer는 고수준 객체 의미를 작게 요약한다. object pointer를 켜면 SA-V val이 64.5에서 68.3, LVOSv2 val이 67.0에서 71.6으로 크게 올랐다. 9 zero-shot 평균은 70.9에서 70.7로 거의 변하지 않아, 어려운 가림과 장기 영상에서 특히 유용한 것으로 해석할 수 있다.

## 8. Memory encoder와 memory bank

### 8.1 Memory encoder

별도의 두 번째 image encoder를 사용하지 않는다. 현재 predicted mask를 convolutional module로 downsample하고, Hiera에서 이미 얻은 **unconditioned** image embedding `E_t`에 element-wise로 더한다. 가벼운 convolution layer가 이를 융합해 memory feature를 만든다.

```text
D_t = MaskDownsampler(M_t)
R_t = MemoryConv(E_t + D_t)
R_t -> projection to 64 channels
```

memory attention으로 이미 객체 조건화된 `Z_t`가 아니라 원래 `E_t`와 mask를 결합한다는 점이 중요하다. 객체 정보는 mask가 제공하고, 시각 표현은 shared image encoder에서 재사용한다.

### 8.2 두 종류의 FIFO

memory bank에는 공간 feature가 두 queue로 들어간다.

- 최근 unprompted prediction 최대 `N`개
- 사용자 prompt가 있었던 frame 최대 `M`개

기본 `N=6` past frames다. VOS처럼 첫 프레임 mask 하나만 prompt라면 첫 prompted frame memory는 유지하고 최근 여섯 prediction memory가 FIFO로 교체된다.

최근 N frame에는 temporal position embedding을 넣어 단기 motion 순서를 표현한다. prompted frame에는 temporal encoding을 넣지 않는다. prompt가 실제 추론에서 매우 먼 과거나 미래 프레임에 올 수 있어, 학습의 좁은 시간 범위에 과적합하기 쉽기 때문이다.

논문은 `M prompted frames`라고 정의하지만 본문과 appendix 표에서 M의 구체적 기본값을 명시하지 않는다. 구현을 재현할 때 config의 prompted-memory cap과 eviction 정책을 별도로 기록해야 한다.

### 8.3 GRU가 필요하지 않았던 이유

memory를 GRU에 통과시켜 recurrent state로 압축하는 ablation은 LVOSv2에서만 소폭 이득이고 전반적 개선은 없었다. 직접 feature를 bank에 저장하는 설계가 더 단순하고 빨라 기본값이 됐다. 이는 학습된 recurrent compression보다 bounded attention memory가 충분했다는 실험 결과다.

## 9. 한 프레임의 streaming 추론 의사코드

```python
def propagate_frame(frame, prompts, bank, image_cache):
    # Each frame is encoded once and shared across objects.
    multi_scale = image_cache.get_or_encode(frame, hiera_encoder)
    image_embedding = fpn(multi_scale.stride16, multi_scale.stride32)

    spatial_memory = bank.recent_memories + bank.prompted_memories
    object_tokens = split_to_64d_tokens(bank.object_pointers)

    conditioned = memory_attention(
        query=image_embedding,
        spatial_memory=spatial_memory,
        object_tokens=object_tokens,
    )

    masks, iou_scores, presence, mask_tokens = mask_decoder(
        conditioned,
        prompt_encoder(prompts),
        high_res_skips=(multi_scale.stride4, multi_scale.stride8),
    )

    k = argmax(iou_scores)
    selected_mask = masks[k]
    pointer = mask_tokens[k]

    memory = memory_encoder(image_embedding, selected_mask)
    if presence.is_occluded:
        memory += learned_occlusion_embedding

    if prompts:
        bank.push_prompted(memory, pointer)
    else:
        bank.push_recent_fifo(memory, pointer, max_recent=6)

    return selected_mask, bank
```

offline correction에서는 prompt frame을 기준으로 앞뒤 방향으로 다시 propagate할 수 있다. online 평가는 현재 이후 frame만 수정하고 이전 결과는 바꾸지 않는다.

## 10. Batch 1 tensor shape와 activation memory

다음은 기본 `1024 x 1024`, BF16 또는 FP16 2 bytes, batch 1에 대한 **리뷰어 계산**이다. 논문은 peak memory를 보고하지 않는다. Hiera 각 stage channel 수와 compiler workspace는 제외한다.

### 10.1 주요 tensor

| tensor | shape | 2-byte 크기 |
|---|---:|---:|
| 입력 RGB | `1 x 3 x 1024 x 1024` | 6.00 MiB |
| stride-16 image embedding | `1 x 256 x 64 x 64` | 2.00 MiB |
| 한 spatial memory | `1 x 64 x 64 x 64` | 0.50 MiB |
| 최근 memory 6개 | `6 x 64 x 64 x 64` | 3.00 MiB |
| 한 object pointer | `256` | 0.00049 MiB |
| pointer를 나눈 4 tokens | `4 x 64` | 동일 |
| SAM식 upsample mask feature | `1 x 32 x 256 x 256` | 4.00 MiB |
| full-resolution mask 1개 | `1 x 1 x 1024 x 1024` | 2.00 MiB |

memory bank의 **저장 크기**만 보면 작다. prompted memory가 `M`개 더 있으면 객체당 `0.5 x M MiB`가 추가된다. 예를 들어 recent 6개와 prompted 2개라면 공간 memory는 객체당 4 MiB다. 이 예의 M=2는 계산 예시이지 논문 기본값이 아니다.

### 10.2 Memory attention score의 숨은 working set

현재 query token 수는 `64 x 64 = 4096`이고, 최근 spatial memory 6개의 token 수는 `6 x 4096 = 24576`이다. prompted memory와 pointer를 빼도 한 cross-attention layer의 query-key 쌍은 다음과 같다.

```text
4096 x 24576 = 100,663,296 score positions per head
```

score를 명시적으로 materialize하면 **head 하나당** 2-byte precision에서 약 192 MiB다. 4 memory-attention layers가 순차 실행되므로 네 장이 동시에 살아야 하는 것은 아니지만, unfused 구현은 한 layer만으로도 모바일 peak memory를 크게 만들 수 있다. 실제 head 수와 projection 세부는 이 논문의 표에 명시되지 않았으므로 총 score 메모리를 임의 확정해서는 안 된다.

FlashAttention 계열은 full score를 저장하지 않아 peak memory를 낮추지만 pairwise 연산과 memory read 자체가 사라지는 것은 아니다. A100에서 사용한 efficient kernel이 target NPU에도 있는지 확인해야 한다.

### 10.3 긴 영상의 frame feature cache

논문은 image encoder를 각 frame에 한 번만 실행하고 여러 객체와 interaction에서 공유한다고 설명한다. 모든 프레임의 stride-16 embedding만 device에 보관한다고 단순 가정하면:

```text
2 MiB/frame x 300 frames = 600 MiB
```

여기에 stride 4, 8 decoder skip feature까지 보관하면 훨씬 커진다. 반대로 streaming 중 최근 frame만 device에 남기고 나머지를 CPU/disk에 offload하거나 correction 시 re-encode하면 resident memory는 줄지만 latency가 늘어난다. 논문은 전체 video feature cache의 resident policy와 peak memory를 보고하지 않는다.

따라서 `bounded memory bank`가 곧 `전체 앱 메모리가 video 길이와 무관하다`는 뜻은 아니다. 객체 memory는 bounded지만 interactive replay를 위한 frame/image cache는 구현 정책에 달려 있다.

### 10.4 여러 객체

image encoder feature는 공유하지만 memory bank, memory attention, mask decoder는 객체마다 독립 실행된다. 객체 수를 `K`라 하면 대략 다음처럼 scaling한다.

```text
shared: image encoding and frame feature cache
per object: 6 recent memories + prompted memories + pointers
            + memory attention + mask decoder + output masks
```

공간 memory 저장만 객체당 3 MiB라면 20개 객체에서 60 MiB다. 더 큰 문제는 memory attention과 decoder compute가 20번 반복된다는 점이다. 논문도 inter-object communication 부재와 효율 개선을 한계로 든다.

## 11. 학습 방식

### 11.1 이미지 pre-training

먼저 SA-1B static images로 pre-train한다.

- 입력: square `1024 x 1024`
- precision: bfloat16
- steps: 약 90k
- batch size: 256
- optimizer: AdamW, beta1=0.9, beta2=0.999
- gradient clipping: L2 max 0.1
- weight decay: 0.1
- learning rate: `4e-4`
- schedule: reciprocal square-root, timescale 1000
- warmup 1k, cooldown 5k
- horizontal flip augmentation
- 이미지당 최대 64 masks
- correction clicks: 7

Hiera는 MAE pretrained weight로 초기화한다. 이미지 90% 이상을 덮는 mask는 SAM과 마찬가지로 필터링한다.

SAM과 달라진 loss/interaction 세부는 다음과 같다.

- IoU prediction에 MSE가 아니라 L1 loss 사용
- IoU logits에 sigmoid를 적용해 0과 1 사이로 제한
- multi-mask 첫 click에서 모든 후보의 IoU head를 supervision
- mask logits는 segmentation loss가 가장 작은 후보 하나만 supervision
- mask-only refinement iteration 두 번을 제거
- correction clicks는 SAM의 8개 대신 7개

### 11.2 Video와 image joint training

pre-training 뒤 SA-V, Internal, SA-1B 10% subset과 선택적으로 DAVIS, MOSE, YouTubeVOS를 섞는다. 각 iteration은 image batch 또는 video batch 하나를 data source 크기에 비례해 선택한다. 서로 다른 batch 구조를 억지로 한 batch에 섞지 않아 compute utilization을 높인다.

기본 full training 설정:

- steps: 약 150k
- resolution: 1024
- precision: bfloat16
- image encoder learning rate: `6e-5`
- other modules learning rate: `3e-4`
- cosine schedule, warmup 7.5k
- batch size: 256
- video sequence length: 8 frames
- sequence당 최대 3 masklets
- prompt frame: 첫 frame을 포함해 최대 2 frames
- correction clicks: 7
- temporal order reversal probability: 0.5
- prediction과 무관하게 GT에서 correction point를 뽑을 확률: 0.1

초기 prompt 확률은 다음과 같다.

```text
ground-truth mask: 0.50
one positive click: 0.25
bounding box: 0.25
```

video augmentation은 horizontal flip, affine rotation 25도, shear 20, color jitter, grayscale 0.05를 포함한다. 0.1 확률의 2x2 mosaic는 같은 영상을 네 quadrant에 복제하고 한 quadrant의 masklet만 target으로 골라, 똑같이 생긴 distractor 중 motion/temporal continuity로 identity를 찾게 한다. 객체 크기도 절반이 되어 small-object 학습 효과가 있다.

### 11.3 Loss

full training loss 가중치는 다음과 같다.

```text
L = 20 * L_focal(mask)
  +  1 * L_dice(mask)
  +  1 * L1(predicted_IoU, true_IoU)
  +  1 * L_CE(object_presence)
```

multi-mask에서는 focal+dice가 가장 작은 후보만 mask supervision을 받지만 모든 후보의 IoU prediction을 학습한다. 정답 mask가 없는 occluded frame에서는 mask loss를 주지 않고 presence head는 항상 학습한다.

### 11.4 16-frame fine-tuning

8-frame 학습과 실제 긴 영상 사이 간극을 줄이기 위해 가장 많은 frame을 사람이 수정한 상위 50% 어려운 masklet만 골라 16-frame sequence로 50k iterations fine-tuning한다.

- 원 schedule의 1/3 길이
- learning rate 절반
- 80 GB A100에 맞추기 위해 image encoder freeze
- open-source VOS dataset은 전체 유지

16 frame조차 장기 영상보다 짧고, 80 GB GPU가 필요하다는 사실은 video training의 activation 부담을 보여준다.

## 12. SA-V data engine

### 12.1 Phase 1: SAM per frame

SAM으로 매 frame을 처음부터 주석하고 brush/eraser로 수정한다. 6 FPS sampling에서 frame당 평균 37.8초, clicked frame당 4.80 clicks가 들었다. 1.4K videos에서 16K masklets를 수집했다.

### 12.2 Phase 2: SAM + SAM 2 Mask

첫 frame spatial mask는 SAM으로 만들고 mask prompt만 받는 중간 SAM 2가 나머지 frame으로 전파한다. 틀린 중간 frame은 다시 마스크를 처음부터 만든다. frame당 7.4초, 전체 frame의 23.25%를 수정하고 수정 frame당 3.61 clicks가 들었다. 63.5K masklets를 수집했다.

### 12.3 Phase 3: full SAM 2

point와 mask를 모두 받고 temporal memory를 사용하는 SAM 2를 annotation loop에 넣는다. 중간 frame을 처음부터 재주석하지 않고 correction click만 추가한다. frame당 4.5초, 수정 frame 19.04%, 수정 frame당 2.68 clicks로 줄었다. Phase 1보다 약 8.4배 빠르며 197.0K masklets를 수집했다.

별도 annotator가 masklet 전체의 일관성을 satisfactory/unsatisfactory로 검증한다. 잘 정의되지 않은 객체는 버리고, 잘 정의됐지만 틀린 masklet은 다시 correction pipeline으로 보낸다.

### 12.4 Automatic masklets

첫 frame에 regular point grid를 넣어 후보 masklet을 자동 생성한다. 검증자가 satisfactory로 판단하면 데이터에 넣고, unsatisfactory failure case 일부는 Phase 3 주석자에게 보내 수정한다. salient object뿐 아니라 배경의 다양한 크기와 위치도 늘리는 목적이다.

## 13. SA-V dataset

### 13.1 규모

| 항목 | SA-V 전체 |
|---|---:|
| videos | 50.9K |
| frames | 4.2M |
| masklets | 642.6K |
| masks | 35.5M |
| manual masklets | 190.9K |
| automatic masklets | 451.7K |
| 평균 video 길이 | 14초 |

기존 최대 VOS dataset보다 masks가 53배 많고 automatic을 빼도 15배 많다. manual masklet의 42.5%는 객체가 사라졌다가 다시 나타나는 구간을 포함한다.

SA-V val은 155 videos, 293 masklets이고 test는 150 videos, 278 masklets다. fast motion, 복잡한 occlusion, disappearance/reappearance를 우선 선정하고, 평가 bias를 줄이기 위해 Phase 1의 per-frame 방식으로 고품질 주석했다.

### 13.2 데이터 추가 효과

동일 iteration 수에서 training data를 단계별로 추가한 결과 J&F는 다음처럼 변한다.

| training data 누적 | SA-V val | 9 zero-shot |
|---|---:|---:|
| VOS + SA-1B | 50.0 | 62.5 |
| + Phase 1 | 53.0 | 66.9 |
| + Phase 2 | 58.8 | 70.9 |
| + Phase 3 | 62.5 | 71.2 |
| + Auto | 63.2 | 71.5 |

각 단계가 in-domain뿐 아니라 zero-shot 평균도 올렸다. 가장 많이 수정된 50K masklets만 고르면 random 50K보다 SA-V val이 63.7에서 66.2로 올라, annotation effort가 hard-example score로 쓰일 수 있음을 보여준다. 하지만 전체 manual 190K의 69.9보다는 낮다.

영상은 region IoU `J`와 boundary accuracy `F`의 평균인 J&F를 주로 사용한다. 이미지는 mIoU다. 아래는 논문 보고값이다.

### 14.1 첫 frame prompt만 사용하는 17 video dataset 평균

| 모델 | 1 click | 3 clicks | 5 clicks | box | GT mask |
|---|---:|---:|---:|---:|---:|

9개 interactive benchmark의 offline/online 평가에서도 SAM 2가 우세했고, 더 높은 정확도를 3배보다 적은 interaction으로 달성했다고 보고한다. 논문 Figure 5의 곡선은 frame 선택 방식이 다른 offline과 online을 구분하므로 하나의 숫자로 합치면 안 된다.

### 14.2 Image SA task, 37 datasets

| 모델 | 학습 데이터 | SA-23 1-click | SA-23 5-click | 14 new video 1-click | FPS |

SA-1B만 사용해도 Hiera-B+는 원본 SAM보다 SA-23 1-click이 0.8 높고 약 6배 빠르다. video data를 섞으면 특히 video-derived image benchmarks에서 큰 폭으로 오른다.

## 15. 모델 ablation

대부분 512 입력의 축소 설정이므로 최종 1024 결과와 직접 섞지 않아야 한다.

### 15.1 해상도

| resolution | MOSE | SA-V val | 9 zero-shot | relative speed | SA-23 mIoU |
|---:|---:|---:|---:|---:|---:|
| 512 | 73.0 | 68.3 | 70.7 | 1.00x | 59.7 |
| 768 | 76.1 | 71.1 | 72.5 | 0.43x | 61.0 |
| 1024 | 77.0 | 70.1 | 72.3 | 0.22x | 61.5 |

1024는 512보다 상대 throughput이 0.22로 약 4.5배 느리지만 모든 video metric에서 일관되게 최고인 것은 아니다. 최종 모델은 전반적 성능과 image task를 고려해 1024를 쓴다. 온디바이스에서는 512/768이 매우 중요한 operating point다.

### 15.2 sequence와 memory

- 학습 frames 4 -> 8: SA-V val 60.0 -> 68.3
- 8 -> 10: SA-V val 68.3 -> 68.1, MOSE 73.0 -> 74.5
- memory 4/6/8: SA-V val 68.6/68.3/69.0, speed 1.01/1.00/0.93
- memory channels 64 -> 256: SA-V val 68.3 -> 66.4, speed 1.00 -> 0.92

64-channel memory가 256-channel보다 저장량을 4배 줄이면서 성능도 나쁘지 않아 기본값이 됐다. memory 수 8의 이득은 작아 최근 6개가 trade-off다.

### 15.3 Memory attention과 encoder scale

memory attention의 self/cross block 수를 `(2,2)`, `(3,2)`, `(4,4)`로 비교하면 `(4,4)`가 SA-V val 68.3으로 가장 높지만 speed는 기준 1.00이다. `(2,2)`는 1.13배 빠르고 SA-V val 67.3이다.

Hiera-S/B+/L의 9 zero-shot은 69.4/70.7/71.9, 상대 speed는 1.33/1.00/0.60이다. 이미지 encoder 확대는 image와 video 모두 개선하지만 memory attention 확대는 주로 video만 개선했다. 모듈별 capacity 효과가 분리된다는 관찰이다.

## 16. 온디바이스 관점의 핵심 병목

### 16.1 논문에 모바일 실측은 없다

논문이 말하는 real-time과 6x speedup은 A100 기준이다. 실제 mobile SoC, Core ML, NNAPI, TFLite, ExecuTorch, Snapdragon NPU에서 latency, peak memory, power를 보고하지 않는다. 따라서 SAM 2를 그대로 온디바이스 가능 모델로 분류할 근거는 부족하다.

### 16.2 Operator 위험 요소

- Hiera window attention과 일부 global attention
- LayerNorm, reshape, window partition/reverse
- memory attention의 긴 cross-attention KV
- 2D RoPE와 object-pointer 예외 처리
- FPN upsample/add
- stride 4, 8 high-resolution skip
- transposed convolution과 dynamic mask dot product
- FIFO bank update, prompted/recent memory selection
- 여러 객체에서 반복되는 decoder와 memory attention

A100에서는 RPB를 제거하고 FlashAttention-2를 활용하지만 모바일 backend에 같은 fused kernel이 없으면 100M 이상 score positions의 working set이 materialize될 수 있다.

### 16.3 권장 모바일 분해

```text
shared per-frame path
  -> reduced-resolution Hiera/student encoder
  -> cache only required feature levels

per-object path
  -> bounded memory selection
  -> compressed/sparse memory attention
  -> prompt/mask decoder

policy layer
  -> detector every frame
  -> SAM 2 only for selected objects or events
  -> old frame features offload/recompute
```

카메라 모든 객체를 SAM 2로 독립 처리하기보다 detector/tracker가 후보를 줄이고, 사용자가 선택한 소수 객체만 memory bank를 유지하는 편이 현실적이다.

### 16.4 실험해야 할 압축 축

1. 1024, 768, 512의 accuracy/latency/energy curve
2. Hiera-B+를 mobile CNN/ViT student로 distillation
3. memory channel 64 -> 32 또는 INT8
4. six-frame dense memory 중 key-frame selection
5. spatial memory token pooling 또는 ROI tokenization
6. memory attention의 linear/sparse attention 대체
7. image encoder INT8, memory/decoder FP16 mixed precision
8. 여러 객체의 shared memory attention 또는 joint decoding
9. prompted frames를 device/host 중 어디에 둘지 cache policy

### 16.5 측정 항목

- frame encoder, memory attention, decoder, memory encoder 각각의 p50/p95
- first prompt와 steady propagation latency
- 객체 1/5/10/20개에서 FPS와 peak memory
- video 30/300/3000 frames에서 resident memory
- memory bank N과 prompted M 변화
- correction 후 forward/backward re-propagation 시간
- operator fallback과 device-host copies
- power, temperature, 10분 이상 thermal throttling
- mask J&F뿐 아니라 identity switch, recovery clicks, time-to-correct

## 17. 한계와 비판적 해석

### 17.1 Shot change와 장기 가림

논문은 shot change, crowded scene, 긴 occlusion, 긴 video에서 객체를 잃거나 혼동할 수 있다고 밝힌다. 사용자가 임의 프레임에서 correction할 수 있다는 것은 recovery mechanism이지 자동 robustness의 완전한 해결은 아니다.

### 17.2 가늘고 빠른 구조

빠르게 움직이는 얇은 객체와 fine detail을 정확히 추적하기 어렵다. stride-16 memory와 64-channel compression, blur가 동시에 작용할 수 있다. stride 4/8 skip은 현재 frame 경계를 살리지만 과거 identity 정보의 공간 해상도 자체를 높이지는 않는다.

### 17.3 비슷한 객체와 명시적 motion 부재

동일한 공 여러 개처럼 appearance가 비슷한 객체에서는 혼동한다. 2x2 mosaic augmentation이 이를 완화하지만 optical flow나 별도 motion state를 명시적으로 모델링하지 않는다. 저자도 explicit motion modeling을 미래 방향으로 제시한다.

### 17.4 객체별 독립 처리

여러 객체가 서로 겹칠 때 상호 배타성이나 공동 문맥을 쓰지 않는다. 한 객체의 마스크가 다른 객체와 중복되어도 각 bank는 이를 모른다. 계산량도 객체 수에 선형으로 늘어난다.

### 17.5 Memory error accumulation

현재 predicted mask가 다음 memory의 입력이다. 잘못된 multimask 후보나 presence 판단이 bank에 들어가면 이후 attention이 그 오류를 강화할 수 있다. predicted IoU와 correction prompt가 완화하지만 불확실성 전체를 memory에 보존하지는 않는다.

### 17.6 데이터 engine의 사람 의존성

automatic masklet도 사람이 satisfactory인지 검증한다. challenging frame 선택과 품질 검증의 자동화가 아직 남아 있다. SA-V test는 어려운 target을 의도적으로 고른 만큼 자연 발생 분포의 평균 성능과는 다른 benchmark다.

### 17.7 공개 데이터와 최종 학습 mix

논문 실험에는 62.9K videos의 licensed Internal dataset이 포함된다. 공개 model은 SA-V manual + Internal + SA-1B로 학습됐다고 적혀 있어 공개 데이터만으로 같은 checkpoint를 완전히 재현하기 어렵다. open-source VOS datasets를 포함한 표와 released model recipe도 구분해야 한다.

### 17.8 속도 비교의 batch 차이

image 130.1 FPS는 batch 10, video 43.8 FPS는 batch 1이다. 두 숫자의 차이를 memory module overhead로 단순 해석할 수 없다. 모바일 연구에서는 모든 모델을 batch 1, 동일 precision, 동일 compiler와 pre/postprocessing으로 재측정해야 한다.

## 18. 강점

1. **통합 모델**: image prompt segmentation과 video propagation/correction을 하나의 architecture로 처리한다.
2. **Streaming memory**: 무한 video 전체를 attention context로 넣지 않고 bounded recent memory와 prompted memory를 분리한다.
3. **공간 memory + semantic pointer**: 경계/위치와 고수준 identity를 서로 다른 표현으로 보존한다.
4. **Occlusion output**: 객체가 존재하지 않는 frame을 명시적으로 모델링한다.
5. **Data engine 공동 진화**: 모델이 annotation을 빠르게 하고, 어려운 annotation이 다시 모델을 개선한다.
6. **폭넓은 zero-shot 검증**: 17 video datasets, 37 image datasets와 interactive/semi-supervised protocol을 포함한다.
7. **세밀한 ablation**: 해상도, sequence length, memory 수와 channel, position encoding, pointer, data mix를 분리한다.

## 20. 로드맵에서의 위치

SAM이 이미지 embedding cache를 만들고 EdgeSAM이 그 encoder를 모바일 CNN으로 줄였다면, SAM 2는 cache 개념을 시간축으로 확장한다.

```text
SAM:
  one image embedding + repeated spatial prompts

SAM 2:
  per-frame embeddings
  + bounded spatial memories
  + semantic object pointers
  + prompts on arbitrary time steps
```

동시에 새로운 온디바이스 병목도 생긴다. encoder만 줄여서는 충분하지 않고 memory attention의 긴 KV, video feature cache, 객체별 반복 실행을 함께 줄여야 한다. 다음 논문인 EdgeTAM류가 image encoder뿐 아니라 memory attention을 모바일 병목으로 다루는 이유다.

통합 시스템에서는 모든 camera object를 SAM 2로 추적하기보다 다음 구조가 합리적이다.

```text
light detector on every frame
  -> selected event/object only
  -> compressed SAM 2 memory pipeline
  -> mask-based ROI/token reduction
  -> VLM request
```

이때 평가 단위는 단일 frame mIoU만이 아니라 긴 occlusion 후 recovery, 필요한 correction clicks, 객체 수별 p95 latency, memory와 thermal steady state여야 한다.

## 21. 핵심 정리

- SAM 2는 현재 frame feature를 memory bank에 cross-attention하고, 예측 mask를 다시 memory로 쓰는 streaming prompt segmenter다.
- spatial memory는 64 channels, 최근 frame 기본 6개이며 object pointer는 256차원을 4개의 64차원 token으로 나눠 쓴다.
- Hiera stride 16/32 feature는 FPN과 memory attention에, stride 4/8 feature는 decoder skip에 사용한다.
- presence head가 occlusion을 모델링하고, multimask 중 최고 predicted-IoU 후보만 다음 frame으로 propagate한다.
- SA-V는 50.9K videos, 642.6K masklets, 35.5M masks이며 SAM 2 data engine은 per-frame SAM보다 annotation을 8.4배 빠르게 했다.
- Hiera-B+는 A100 batch 1 video에서 43.8 FPS지만 모바일 latency와 peak memory는 미보고다.
- six-frame spatial bank 저장은 객체당 약 3 MiB지만 dense cross-attention은 layer당 head 하나 기준 100.7M score positions를 만들 수 있어 working memory가 핵심이다.
- 전체 frame cache와 여러 객체의 독립 memory/decoder가 실제 장치에서 추가 병목이다.
- 온디바이스 후속 연구는 encoder 경량화뿐 아니라 memory selection, token compression, cache policy, multi-object sharing까지 함께 다뤄야 한다.
